> 日期：2026-07-08

## 一、流程架构

`
[tasks.json 输入]
     |
     v
+----------------------------------+
| 阶段一：需求定义                  |
|  - L1: 苏格拉底询问 -> 用户确认  |
|  - L2/L3: 自动解析并锁定         |
|  输出: 结构化需求文档             |
+----------------------------------+
           |
           v
+----------------------------------+
| 阶段二：策略制定                  |
|  - 框架生成 1-3 个策略           |
|  - 每个策略含原子测试+评分体系    |
|  - >=2 个策略有实质差异的门禁     |
|  - L1: 用户确认策略               |
|  - L2/L3: 框架自动选最优         |
|  输出: 选定策略+测试清单+预算    |
+----------------------------------+
           |
           v
+----------------------------------+
| 阶段三：工程实现（核心循环）     |
|  循环体:                         |
|  1. GLM 工作调用                  |
|  2. 完成度评估                    |
|     L1: 评估调用(GLM)            |
|     L2: 规则评估->瓶颈时切GLM    |
|     L3: 全程规则评估             |
|  3. 边际收益计算:本轮-上轮       |
|  4. 决策:                        |
|     >= 20% -> 继续                 |
|     10-20% -> 继续，降低评估频率 |
|     5-10% -> 标记风险，准备切换   |
|     < 5% -> 切换策略              |
|     连续3轮 < 10% -> 强制停止     |
|  5. 所有策略失效->暂停，通知用户  |
|  6. 测试全通过->完成              |
+----------------------------------+
           |
           v
+----------------------------------+
| 额度管理和任务切换              |
|  - 5h/1000次调用                 |
|  - 单个任务预算<=剩余额度40%     |
|  - 额度耗尽->驻留轮询待机         |
|  - 恢复后按剩余完成度降序恢复    |
|  - 完成一个->自动切下一个         |
|  - 全部完成->汇总报告             |
+----------------------------------+
`

## 二、核心机制

### 任务层级

| 层级 | 描述 | 需求确认 | 策略确认 | 评估方式 |
|---|---|---|---|---|
| L1 战略 | 新功能设计、架构变更 | 苏格拉底询问后用户确认 | 用户确认 | GLM 评估调用 |
| L2 战术 | Bug修复、性能优化 | 自动 | 自动 | 规则评估->瓶颈时切GLM |
| L3 执行 | 重构、文档、依赖更新 | 自动 | 自动 | 全程规则评估 |

### 评估体系

总分 = sum(单项得分 * 单项权重) / sum(总权重)

| 总分范围 | 等级 | 含义 |
|---|---|---|
| >= 90 | S | 优质实现 |
| >= 80 | A | 良好实现 |
| >= 60 | B | 基本可用 |
| < 60 | F | 不及格 |

### 边际收益阈值

| 进步量 | 决策 |
|---|---|
| >= 20% | 继续当前策略 |
| 10-20% | 继续，降低评估频率 |
| 5-10% | 标记风险，准备换策略 |
| < 5% | 切换策略 |
| 连续3轮 < 10% | 强制停止 |

### 完成度度量

| 任务类型 | 维度及权重 |
|---|---|
| 代码/工程 | 测试通过率(50%)+边界覆盖(30%)+编译正确(20%) |
| 文档/分析 | 结构完整(40%)+信息准确(40%)+格式(20%) |
| 通用问答 | 目标覆盖(60%)+深度指数(40%) |

## 三、用户参与节点

| 节点 | 参与方式 |
|---|---|
| L1 需求定义 | 苏格拉底询问后用户确认 |
| L2/L3 需求定义 | 自动，不介入 |
| L1 策略选择 | 用户确认策略 |
| L2/L3 策略 | 自动 |
| 实现阶段 | 不介入 |
| 所有策略均失败 | 通知用户 |
| 全部任务完成 | 通知用户 |
