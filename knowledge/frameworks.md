 # 主流量化回测框架对比
 
 > 对比范围：Qlib (Microsoft)、vnpy、Backtrader、Zipline
 
 ---
 
 ## Qlib (Microsoft)
 
 | 属性 | 内容 |
 |------|------|
 | GitHub Stars | ~16k |
 | 语言 | Python |
 | 核心定位 | AI 量化研究平台 |
 
 ### 优点
 1. **AI 原生设计**：内置 Alpha 因子挖掘、模型训练、回测、部署全链路，对 ML/DL 策略支持最好
 2. **高性能**：基于 pandas + numpy 向量化运算，回测速度快
 3. **数据集市**：内置数据下载和缓存管理，减少数据工程工作
 4. **模型回测框架完善**：支持滚动回测、交叉验证、模型选择等高级功能
 5. **在线推理**：支持模型部署到生产环境
 
 ### 缺点
 1. **学习曲线陡峭**：架构复杂，文档以 API 为主，新手入门困难
 2. **过度强调 AI**：对传统规则策略支持较弱，自定义策略需要较多适配工作
 3. **数据适配局限**：主要支持 A 股和美股的标准化数据格式，DIY 数据源需要开发 adapter
 4. **社区较小**：国内用户为主，英文资料有限
 
 ### 适用场景
 - AI/ML 驱动的因子挖掘和策略研究
 - 机构级的量化研究团队
 - 需要从研究到生产一体化的场景

 ## vnpy
 
 | 属性 | 内容 |
 |------|------|
 | GitHub Stars | ~26k |
 | 语言 | Python |
 | 核心定位 | 全栈量化交易系统（含实盘） |
 
 ### 优点
 1. **实盘交易**：支持 CTP、XTP、IB 等数十家柜台接口，是国内实盘接入最完善的框架
 2. **全功能架构**：从行情、回测到实盘、风控、绩效分析一站式覆盖
 3. **中文生态**：中文文档完善，国内社区活跃
 4. **模块化**：组件间低耦合，可根据需要只使用回测模块
 5. **事件驱动**：回测引擎基于事件驱动，更贴近真实交易
 
 ### 缺点
 1. **回测速度**：事件驱动架构在大规模回测时比向量化慢
 2. **AI 支持弱**：不内置 AI/ML 模型支持，需要自行集成
 3. **安装复杂**：依赖较多（MongoDB、RabbitMQ 等可选组件）
 4. **文档质量参差**：部分模块文档滞后于代码更新
 
 ### 适用场景
 - 国内期货/股票实盘交易
 - 需要 CTP 等专业柜台的机构交易
 - 中低频策略的全流程系统搭建

 ## Backtrader
 
 | 属性 | 内容 |
 |------|------|
 | GitHub Stars | ~15k |
 | 语言 | Python |
 | 核心定位 | 轻量级回测框架 |
 
 ### 优点
 1. **上手简单**：API 设计直观，几分钟内可以跑通第一个回测
 2. **文档丰富**：官方文档 + 大量社区教程和示例
 3. **轻量无依赖**：只依赖 pandas, numpy, matplotlib
 4. **回测到实盘**：设计上回测代码可直接用于实盘（使用 broker 接口）
 5. **策略可组合**：支持多个 data feed、策略、观察者叠加
 6. **自定义能力强**：几乎每个组件（手续费、滑点、订单类型）都可重写
 
 ### 缺点
 1. **性能瓶颈**：纯 Python 事件驱动，100+ 标的会显著变慢
 2. **AI 不支持**：无内置 ML 支持，需手动集成
 3. **多标的支持弱**：组合回测不是设计重点
 4. **维护停滞**：核心作者已较少更新，Issue 响应慢
 5. **因子分析缺位**：无因子 IC/IR 分析等研究工具
 
 ### 适用场景
 - 个人投资者的快速原型验证
 - 单标的/少量标的的策略开发
 - 教学和学习回测基础知识

 ## Zipline
 
 | 属性 | 内容 |
 |------|------|
 | GitHub Stars | ~18k |
 | 语言 | Python |
 | 核心定位 | 事件驱动回测引擎（Quantopian 遗产） |
 
 ### 优点
 1. **Quantopian 传承**：当时最成熟的开源回测引擎，设计经过大规模使用验证
 2. **数据管道完善**：内置 bundle 机制管理历史数据下载和缓存
 3. **订单执行模型**：支持市价单、限价单、止损单等多种订单类型
 4. **绩效分析**：集成 pyfolio 进行详细的绩效归因
 5. **研究环境**：配套 notebook 交互式探索数据
 
 ### 缺点
 1. **维护状态差**：Quantopian 关闭后，Zipline 更新缓慢（Zipline-Reloaded 社区分支稍好）
 2. **安装困难**：依赖 Ta-Lib、pandas-datareader 等在 Windows 上安装繁琐
 3. **数据格式限制**：强制使用 OHLCV 格式，自定义数据源需要写 bundle extension
 4. **性能一般**：事件驱动，大标的速度慢
 5. **A 股支持**：需要大量适配工作（涨跌停、T+1、印花税等）
 
 ### 适用场景
 - 美股策略的回测研究
 - 使用 pyfolio 进行详细绩效分析
 - 教学和学术研究（Quantopian 时代遗存的大量教学资源）

 ## 选择建议
 
 本项目选择 **自研回测框架**（非直接使用上述框架），原因：
 1. 本项目核心创新在于 **Market State Perception + Sentiment Cross-Validation + Meta-Learning** 的 Dual-Agent 架构，现有框架都不直接支持
 2. 需要完全控制数据划分（Purging/Embargo/CPCV）和过拟合检验（PBO/DSR）
 3. 回测规模较小（9 只 ETF），性能不是瓶颈
 4. Backtrader 的 `CostModel` 和策略注入设计可作为架构参考
 5. Qlib 的 AI pipeline 设计思想可参考（但直接使用会受限于其数据格式）
 
 但每个框架的设计理念和可复用组件会在本项目实现中认真参考。
